# 典型工作流

## 内容创作工作流

### 📰 博客文章创作流程

```
┌─────────────────────────────────────────────────────────────┐
│ 阶段 1: 研究 (Perplexity)                                   │
├─────────────────────────────────────────────────────────────┤
│ 1. 搜索最新趋势和数据                                        │
│ 2. 收集引用来源和统计                                        │
│ 3. 分析竞品内容                                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段 2: 大纲 (ChatGPT/Claude)                               │
├─────────────────────────────────────────────────────────────┤
│ 1. 生成文章结构                                              │
│ 2. 确定关键论点                                              │
│ 3. 规划段落逻辑                                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段 3: 初稿 (Claude - 长文本优势)                           │
├─────────────────────────────────────────────────────────────┤
│ 1. 逐段撰写内容                                              │
│ 2. 保持语气一致                                              │
│ 3. 自然融入关键词                                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段 4: 配图 (Midjourney/DALL-E 3)                          │
├─────────────────────────────────────────────────────────────┤
│ 1. 生成封面图                                                │
│ 2. 创建文内插图                                              │
│ 3. 统一视觉风格                                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段 5: 优化 (ChatGPT)                                      │
├─────────────────────────────────────────────────────────────┤
│ 1. SEO 优化检查                                              │
│ 2. 可读性改进                                                │
│ 3. 最终校对                                                  │
└─────────────────────────────────────────────────────────────┘
```

**时间估算：** 2-4 小时 (vs 传统 6-8 小时)

**提示词模板：**
```
# 研究阶段
"搜索关于 [主题] 的最新数据和趋势，提供 5-10 个可靠来源"

# 大纲阶段
"为 [主题] 创建博客大纲，包含：引人注目的标题、5-7 个主要章节、每章要点"

# 写作阶段
"基于以下大纲撰写 2000 字文章，语气 [专业/轻松/幽默]，目标读者 [描述]"

# 配图阶段
"生成 [描述] 的图像，风格 [极简/插画/照片级]，适合科技博客"
```

### 📧 邮件营销工作流

```
研究受众 → 生成主题行 → 撰写正文 → A/B 测试 → 发送优化
   ↓            ↓           ↓          ↓          ↓
Perplexity   ChatGPT     Jasper    ChatGPT    Jasper/
           (10 个选项)  (模板)    (变体)     邮件平台
```

**步骤详解：**

1. **受众研究** (Perplexity)
   - 搜索行业痛点
   - 分析竞品邮件
   - 收集热点话题

2. **主题行生成** (ChatGPT)
   ```
   提示词："为 [产品] 生成 20 个邮件主题行，
   包含：好奇心驱动、紧迫感、个性化元素"
   ```

3. **正文撰写** (Jasper)
   - 选择营销模板
   - 输入产品信息
   - 生成初稿

4. **A/B 变体** (ChatGPT)
   ```
   提示词："创建 3 个版本：
   - 版本 A: 直接销售导向
   - 版本 B: 故事叙述导向
   - 版本 C: 教育价值导向"
   ```

5. **优化发送** (邮件平台 AI)
   - 最佳发送时间
   - 个性化推荐
   - 打开率预测

## 编程开发工作流

### 💻 功能开发流程

```
┌─────────────────────────────────────────────────────────────┐
│ 1. 需求理解 (ChatGPT)                                       │
│    "帮我分析这个需求，列出技术要点和潜在难点"                 │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 2. 架构设计 (Claude - 长上下文)                              │
│    "设计实现方案，包含：模块划分、接口定义、数据流"            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 3. 代码实现 (Cursor)                                        │
│    - 项目上下文理解                                          │
│    - 智能代码补全                                            │
│    - 实时错误检测                                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 4. 代码审查 (ChatGPT + Cursor)                              │
│    "审查这段代码，检查：性能、安全、可维护性"                 │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 5. 文档生成 (ChatGPT)                                       │
│    "为这个模块生成 README 和 API 文档"                        │
└─────────────────────────────────────────────────────────────┘
```

**Cursor 高效用法：**
```
# 项目理解
@workspace 解释这个项目的架构

# 功能实现
@file 实现用户登录功能，包含 JWT 认证

# 代码审查
@selection 这段代码有什么潜在问题？

# Bug 修复
@errors 修复所有 TypeScript 错误
```

│ • 筛选高质量论文                                            │
│ • 收集引用信息                                              │
└─────────────────────────────────────────────────────────────┘
                            ↓

│ 阶段 2: 快速阅读 (Claude - 上传 PDF)                         │
├─────────────────────────────────────────────────────────────┤

│ • 对比不同研究                                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段 3: 笔记整理 (Notion AI)                                │
├─────────────────────────────────────────────────────────────┤
│ • 结构化笔记                                                │
│ • 建立知识关联                                              │
│ • 生成摘要卡片                                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│ 阶段 4: 写作辅助 (ChatGPT/Claude)                           │
├─────────────────────────────────────────────────────────────┤

│ • 段落润色                                                  │

**Claude 论文分析提示词：**
```

3. 方法论的优缺点？
4. 与我研究的关联性？
5. 值得引用的关键点？"
```

## 设计创意工作流

### 🎨 视觉设计流程

```
概念 → 草图 → 生成 → 迭代 → 精修
 ↓      ↓      ↓      ↓      ↓
头脑   描述   Midj   调整   PS/
风暴   细化   ourney 参数   后期
```

**Midjourney 迭代流程：**
```
# 第 1 轮：探索风格
/imagine prompt: [基础描述] --v 6

# 第 2 轮：选择最佳，变体
V1/V2/V3/V4 或 🔄

# 第 3 轮：精细化
 upscale + describe 获取提示词

# 第 4 轮：最终调整
添加参数：--ar 16:9 --q 2 --style raw
```

**提示词结构：**
```
[主体] + [动作/状态] + [环境/背景] + [风格] + [光线] + [色彩] + [参数]

示例：
"A minimalist logo for a tech startup, 
 geometric shapes, clean lines, 
 blue and white color scheme, 
 professional, modern --v 6 --q 2"
```

## 社交媒体运营工作流

### 📱 多平台内容分发

```
┌─────────────────────────────────────────────────────────────┐
│ 核心内容创作 (ChatGPT)                                      │
│ "写一篇关于 [主题] 的 500 字文章"                              │
└─────────────────────────────────────────────────────────────┘
                            ↓
        ┌───────────────────┼───────────────────┐
        ↓                   ↓                   ↓
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│  Twitter      │   │  LinkedIn     │   │  Instagram    │
│  (280 字符)    │   │  (专业长文)    │   │  (视觉 + 短文案)│
└───────────────┘   └───────────────┘   └───────────────┘
        ↓                   ↓                   ↓
  ChatGPT 缩写         ChatGPT 改写        ChatGPT+DALL-E
  "改成推文风格"       "改成专业语气"       "生成配图 + 文案"
```

**平台适配提示词：**
```
# Twitter
"将以下内容改写成 5 条推文线程，每条<280 字符，
包含 emoji 和话题标签"

# LinkedIn
"将这篇文章改写成 LinkedIn 帖子，
专业语气，包含行业洞察和行动号召"

# Instagram
"为这张图片生成 Instagram 文案，
轻松语气，15-20 个相关 hashtag"

# 小红书
"改写为小红书笔记，
包含：吸引眼球标题、emoji、个人体验、实用 tips"
```
