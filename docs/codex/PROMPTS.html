Prompt 编写技巧

核心原则

1. 明确具体 (Be Specific)

❌ 模糊的请求:

写个函数

✅ 具体的请求:

用 Python 写一个函数，接收一个整数列表，返回其中的第二大值。

如果列表长度小于 2，抛出 ValueError。


添加类型注解和文档字符串。

2. 提供上下文 (Provide Context)

❌ 缺少上下文:

修复这个 bug

✅ 完整上下文:

以下 Python 函数在输入空列表时抛出 IndexError，请修复：


def get_max(numbers):


    sorted_nums = sorted(numbers)


    return sorted_nums[-1]



期望行为：空列表返回 None

3. 指定格式 (Specify Format)

❌ 不指定格式:

生成用户模型

✅ 指定格式:

用 TypeScript 创建 User 接口，包含以下字段：

  • id: string (UUID)
    • 

  • email: string
    • 

  • name: string
    • 

  • createdAt: Date
    • 

  • updatedAt: Date
    • 



只输出代码，不要解释。

高级技巧

1. 思维链 (Chain of Thought)

让模型展示思考过程，提高准确性：

请逐步分析以下问题，然后给出答案：


问题：优化这个 O(n²) 的算法到 O(n log n)



[代码]



要求：


1. 先分析当前算法的时间复杂度


2. 指出性能瓶颈


3. 提出优化思路


4. 给出优化后的代码

4. 分步请求 (Step-by-Step)

复杂任务拆分成多个 prompt：

第一步：设计数据库 schema

第二步：生成 ORM 模型


第三步：创建 CRUD 操作


第四步：添加单元测试

5. 约束条件 (Constraints)

明确限制条件提高输出质量：

写一个 API 端点，要求：

  • 使用 FastAPI
    • 

  • 必须包含输入验证
    • 

  • 必须有错误处理
    • 

  • 必须有类型注解
    • 

  • 不能超过 50 行代码
    • 

  • 不使用外部数据库
    •

避免的陷阱

❌ 不要这样做

1. 过于模糊

   帮我写代码

2. 一次性太多

   创建整个电商系统，包括用户、商品、订单、支付...

3. 缺少关键信息

   修复这个错误（没有提供代码或错误信息）

4. 矛盾的要求

   代码要尽可能短，但要包含完整的错误处理和注释

✅ 应该这样做

1. 具体明确

   用 Python 写一个函数，验证邮箱格式

2. 分步进行

   第一步：设计用户数据模型

   第二步：创建用户注册 API

3. 提供完整信息

   环境：Python 3.9, Flask 2.0

   问题：POST 请求返回 405


   代码：[附上代码]

4. 合理的要求

   代码要简洁，但必须包含输入验证和错误处理

Prompt 检查清单

在发送 prompt 前，检查：

  • [ ] 任务描述是否清晰具体？
  • [ ] 是否指定了编程语言和版本？
  • [ ] 是否说明了输入和输出？
  • [ ] 是否包含必要的约束条件？
  • [ ] 是否提供了相关上下文或示例代码？
  • [ ] 是否指定了输出格式？
  • [ ] 是否有矛盾或不合理的要求？
  • [ ] 复杂任务是否已拆分？
