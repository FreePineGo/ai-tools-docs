# 常见问题解答 (FAQ)

## 基础问题

### Q1: Codex 和 GPT-4 有什么区别？

**A**: 
- **Codex** 是 OpenAI 专门针对代码优化的模型，基于 GPT-3 训练，于 2023 年停止独立服务
- **GPT-4** 是更强大的通用模型，代码能力与 Codex 相当甚至更强
- 目前推荐使用 GPT-4 或 GPT-4 Turbo 进行代码生成任务

### Q2: 需要付费吗？费用如何计算？

**A**: 
是的，OpenAI API 是付费服务。费用按 Token 计算：

| 模型 | 输入价格 | 输出价格 |
|------|----------|----------|
| GPT-4 | $0.03/1K tokens | $0.06/1K tokens |
| GPT-4 Turbo | $0.01/1K tokens | $0.03/1K tokens |
| GPT-3.5 Turbo | $0.0005/1K tokens | $0.0015/1K tokens |

1000 tokens ≈ 750 个英文单词或 500-600 行代码

### Q3: 如何获取 API 密钥？

**A**:
1. 访问 [platform.openai.com](https://platform.openai.com)
2. 注册或登录账户
3. 进入 API Keys 页面
4. 点击 "Create new secret key"
5. 复制并安全保存密钥（只显示一次）

### Q4: API 密钥泄露了怎么办？

**A**:
1. **立即**在 OpenAI 平台删除该密钥
2. 检查是否有异常使用记录
3. 生成新密钥
4. 更新所有使用该密钥的地方
5. 考虑设置使用限额和告警

## 使用问题

### Q5: 生成的代码不能运行怎么办？

**A**:
1. **检查错误信息**：仔细阅读报错内容
2. **提供完整上下文**：包括语言版本、依赖库、完整错误堆栈
3. **迭代修复**：将错误信息反馈给 AI，请求修复
4. **人工审查**：AI 生成的代码必须经过测试

示例反馈 prompt：
```
以下代码运行时报错：

[代码]

错误信息：
[完整错误]

环境：Python 3.9, pandas 1.5.0

请分析原因并修复。
```

### Q6: 如何确保生成的代码质量？

4. **性能测试**：验证性能是否符合要求
5. **安全扫描**：检查潜在安全漏洞

## 技术问题

### Q9: 遇到 Rate Limit 怎么办？

**A**:
1. **降低请求频率**：添加请求间隔
2. **实现重试逻辑**：指数退避
3. **升级账户**：提高限额
4. **使用缓存**：避免重复请求

```python
import time
from openai import RateLimitError

def make_request_with_retry(prompt, max_retries=3):
    for i in range(max_retries):
        try:
            return client.chat.completions.create(...)
        except RateLimitError:
            wait_time = 2 ** i  # 指数退避
            time.sleep(wait_time)
    raise Exception("重试失败")
```

### Q10: 如何降低 Token 使用量？

**A**:
1. **精简 prompt**：去掉冗余描述
2. **复用上下文**：避免重复发送相同内容
3. **选择合适模型**：简单任务用 GPT-3.5
4. **限制输出长度**：设置 max_tokens
5. **使用缓存**：相同请求返回缓存结果

### Q11: 可以离线使用吗？

**A**:
不可以。OpenAI API 必须联网使用。

离线替代方案：
- **本地模型**：CodeLlama、StarCoder、WizardCoder
- **本地工具**：Ollama、LM Studio
- **IDE 插件**：Cursor、Codeium（有离线模式）

### Q12: 生成的代码有版权问题吗？

**A**:
- OpenAI 声明用户对生成内容拥有所有权
- 但生成的代码可能与训练数据中的代码相似
- **建议**：
  - 审查生成代码的独特性
  - 避免直接用于商业产品 without 审查
  - 注意第三方库的许可证

## 安全问题

### Q13: 生成的代码安全吗？

**A**:
**不一定**。AI 可能生成：
- 有安全漏洞的代码（SQL 注入、XSS 等）
- 使用过时的、不安全的 API
- 缺少必要的输入验证

**必须**：
1. 人工审查所有生成代码
2. 运行安全扫描工具
3. 遵循安全编码规范
4. 进行渗透测试

### Q14: 会泄露我的代码吗？

**A**:
根据 OpenAI 政策：
- **API 用户数据**：不会用于训练模型
- **企业用户**：有额外的数据保护
- **建议**：
  - 不要发送敏感代码（密钥、密码等）
  - 使用环境变量管理敏感信息
  - 考虑企业版获得更强保护

### Q15: 可以用 AI 写恶意代码吗？

**A**:
**不应该也不能**。
- OpenAI 有使用政策禁止恶意用途
- 生成的恶意代码可能违反法律
- AI 会拒绝明显的恶意请求

## 最佳实践

### Q16: 如何将 AI 集成到开发流程？

**A**:
推荐工作流：
```
1. 需求分析 → AI 辅助设计
2. 架构设计 → AI 生成框架
3. 编码实现 → AI 生成代码片段
4. 代码审查 → AI 辅助审查
5. 测试编写 → AI 生成测试用例
6. 文档编写 → AI 生成文档
```

### Q17: 适合什么场景使用？

**A**:
✅ **推荐场景**：
- 原型开发
- 样板代码生成
- 学习新语言/框架
- 代码重构建议
- 单元测试生成
- 文档编写
- Debug 辅助

❌ **不推荐场景**：
- 安全关键系统（无审查直接使用）
- 完全依赖 AI 写生产代码
- 处理敏感数据
- 需要 100% 准确性的场景

### Q18: 如何提高生成准确率？

**A**:
1. **详细的 prompt**：越具体越好
2. **提供示例**：Few-shot learning
3. **分步请求**：复杂任务拆分
4. **指定约束**：明确限制条件
5. **迭代优化**：多轮对话改进

### Q19: 如何处理长代码文件？

**A**:
1. **分段处理**：按函数/类拆分
2. **提供上下文**：发送相关部分
3. **使用文件上传**：GPT-4 支持文件
4. **摘要 + 详情**：先获取整体理解

```
这个文件有 1000 行，我先发前 200 行，
请分析整体结构，然后我会发送具体要修改的部分。
```

## 故障排查

### Q21: 常见错误及解决方案

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Invalid API key` | 密钥错误或过期 | 检查密钥，重新生成 |
| `Rate limit exceeded` | 请求太频繁 | 降低频率，实现重试 |
| `Insufficient quota` | 余额不足 | 充值或升级 |
| `Model not found` | 模型名称错误 | 检查模型名称 |
| `Context length exceeded` | 输入太长 | 缩短 prompt 或分段 |
| `Timeout` | 网络或服务器问题 | 重试，检查网络 |

### Q22: 输出被截断怎么办？

**A**:
1. **增加 max_tokens**：提高输出限制
2. **分段请求**：要求分多次输出
3. **使用 continue**：请求继续生成

```
请继续上面的代码，从第 50 行开始
```

### Q23: 模型不理解我的需求怎么办？

**A**:
1. **换种说法**：重新组织语言
2. **提供示例**：给出期望的输出示例
3. **分步引导**：先确认理解再执行
4. **指定角色**：设定专家角色

```
你是一位有 10 年经验的 Python 专家，请...
```
