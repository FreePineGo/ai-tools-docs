# Prompt 编写技巧

## 核心原则

### 1. 明确具体 (Be Specific)

❌ **模糊的请求**:
```
写个函数
```

✅ **具体的请求**:
```
用 Python 写一个函数，接收一个整数列表，返回其中的第二大值。
如果列表长度小于 2，抛出 ValueError。
添加类型注解和文档字符串。
```

### 2. 提供上下文 (Provide Context)

❌ **缺少上下文**:
```
修复这个 bug
```

✅ **完整上下文**:
```
以下 Python 函数在输入空列表时抛出 IndexError，请修复：

def get_max(numbers):
    sorted_nums = sorted(numbers)
    return sorted_nums[-1]

期望行为：空列表返回 None
```

### 3. 指定格式 (Specify Format)

❌ **不指定格式**:
```
生成用户模型
```

✅ **指定格式**:
```
用 TypeScript 创建 User 接口，包含以下字段：
- id: string (UUID)
- email: string
- name: string
- createdAt: Date
- updatedAt: Date

只输出代码，不要解释。
```

## Prompt 模板

### 代码生成模板

```
角色：[资深 Python 工程师 / React 专家 / SQL 专家等]
任务：[具体要完成的任务]
语言：[编程语言]
要求：
- [要求 1]
- [要求 2]
- [要求 3]
约束：[任何限制条件]
输出格式：[代码/解释/两者都要]
```

**示例**:
```
角色：资深 Python 后端工程师
任务：创建 JWT 认证装饰器
语言：Python 3.9+
要求：
- 验证 JWT token 有效性
- 从 token 中提取用户 ID
- 将用户 ID 添加到请求上下文
- 处理过期和无效 token
约束：使用 PyJWT 库
输出格式：只输出代码，包含类型注解
```

### 代码审查模板

```
请审查以下代码：

[代码内容]

审查要点：
1. 潜在 bug
2. 性能问题
3. 安全漏洞
4. 代码风格
5. 改进建议

请按以上要点逐一分析，并提供修改后的代码。
```

### Debug 模板

```
问题：[描述问题现象]
环境：[语言版本、框架版本等]
预期行为：[应该发生什么]
实际行为：[实际发生了什么]
错误信息：[完整的错误堆栈]

相关代码：
[代码内容]

已尝试的解决方案：
- [尝试 1]
- [尝试 2]

请分析原因并提供解决方案。
```

### 重构模板

```
请重构以下代码以改善：
- 可读性
- 可维护性
- 性能

原始代码：
[代码内容]

约束条件：
- 保持现有功能不变
- 不引入新的外部依赖
- 遵循 [PEP 8 / Airbnb 等] 风格指南

请提供：
1. 重构后的代码
2. 主要改进说明
```

## 高级技巧

### 1. 思维链 (Chain of Thought)

让模型展示思考过程，提高准确性：

```
请逐步分析以下问题，然后给出答案：

问题：优化这个 O(n²) 的算法到 O(n log n)

[代码]

要求：
1. 先分析当前算法的时间复杂度
2. 指出性能瓶颈
3. 提出优化思路
4. 给出优化后的代码
```

### 2. 少样本学习 (Few-Shot Learning)

提供示例帮助模型理解期望输出：

```
按照以下示例的格式生成代码：

示例 1:
输入：创建加法函数
输出：
```python
def add(a, b):
    return a + b
```

示例 2:
输入：创建乘法函数
输出：
```python
def multiply(a, b):
    return a * b
```

现在：
输入：创建除法函数，处理除零错误
输出：
```

### 3. 角色设定 (Role Playing)

```
你是一位有 10 年经验的安全工程师，专注于代码安全审计。
请审查以下代码中的安全漏洞：

[代码]

请按严重程度排序，并提供修复方案。
```

### 4. 分步请求 (Step-by-Step)

复杂任务拆分成多个 prompt：

```
第一步：设计数据库 schema
第二步：生成 ORM 模型
第三步：创建 CRUD 操作
第四步：添加单元测试
```

### 5. 约束条件 (Constraints)

明确限制条件提高输出质量：

```
写一个 API 端点，要求：
- 使用 FastAPI
- 必须包含输入验证
- 必须有错误处理
- 必须有类型注解
- 不能超过 50 行代码
- 不使用外部数据库
```

## 常见场景 Prompt

### API 开发

```
创建 RESTful API 端点：
- 框架：[FastAPI/Express/Spring 等]
- 功能：[CRUD/认证/文件上传等]
- 认证：[JWT/OAuth2/无]
- 数据库：[PostgreSQL/MongoDB 等]
- 需要：[验证/日志/缓存/限流]

输出完整的端点代码和测试示例。
```

### 数据处理

```
编写数据处理脚本：
- 输入：[CSV/JSON/数据库等]
- 处理：[清洗/转换/聚合/分析]
- 输出：[格式和位置]
- 库偏好：[pandas/d3 等]
- 性能要求：[内存/速度限制]
```

### 测试生成

```
为以下代码生成单元测试：

[代码]

要求：
- 测试框架：[pytest/Jest/JUnit 等]
- 覆盖率目标：[行覆盖/分支覆盖]
- 包含：[正常情况/边界情况/错误情况]
- Mock：[需要 mock 的外部依赖]
```

### 文档生成

```
为以下代码生成文档：

[代码]

要求：
- 文档风格：[Google/NumPy/JSDoc 等]
- 包含：[参数说明/返回值/异常/示例]
- 语言：[中文/英文]
- 格式：[docstring/独立文档]
```

### 代码转换

```
将以下 [源语言] 代码转换为 [目标语言]：

[源代码]

要求：
- 保持功能完全一致
- 使用目标语言的最佳实践
- 添加必要的注释
- 处理语言特定的边界情况
```

## 优化技巧

### 1. 温度参数调整

```
# 确定性任务（代码生成）
temperature = 0.1-0.3

# 创意任务（架构设计）
temperature = 0.5-0.7

# 探索性任务（头脑风暴）
temperature = 0.7-0.9
```

### 2. 系统消息优化

```python
# 通用系统消息
system_prompt = """你是资深软件工程师。
- 只输出高质量、可运行的代码
- 遵循最佳实践和设计模式
- 添加必要的错误处理
- 代码要有良好的可读性
- 使用清晰的变量命名"""

# 特定领域系统消息
system_prompt = """你是 Python 数据科学专家。
- 使用 pandas 和 numpy
- 代码要向量化，避免循环
- 添加类型注解
- 包含性能优化"""
```

### 3. 输出控制

```
在 prompt 末尾添加：

"只输出代码，不要解释"
"先解释思路，再给代码"
"用中文注释"
"输出 Markdown 格式的代码块"
```

### 4. 迭代优化

```
第一轮：生成基础代码
第二轮：添加错误处理
第三轮：优化性能
第四轮：添加测试
```

## 避免的陷阱

### ❌ 不要这样做

1. **过于模糊**
   ```
   帮我写代码
   ```

2. **一次性太多**
   ```
   创建整个电商系统，包括用户、商品、订单、支付...
   ```

3. **缺少关键信息**
   ```
   修复这个错误（没有提供代码或错误信息）
   ```

4. **矛盾的要求**
   ```
   代码要尽可能短，但要包含完整的错误处理和注释
   ```

### ✅ 应该这样做

1. **具体明确**
   ```
   用 Python 写一个函数，验证邮箱格式
   ```

2. **分步进行**
   ```
   第一步：设计用户数据模型
   第二步：创建用户注册 API
   ```

3. **提供完整信息**
   ```
   环境：Python 3.9, Flask 2.0
   问题：POST 请求返回 405
   代码：[附上代码]
   ```

4. **合理的要求**
   ```
   代码要简洁，但必须包含输入验证和错误处理
   ```

## Prompt 检查清单

在发送 prompt 前，检查：

- [ ] 任务描述是否清晰具体？
- [ ] 是否指定了编程语言和版本？
- [ ] 是否说明了输入和输出？
- [ ] 是否包含必要的约束条件？
- [ ] 是否提供了相关上下文或示例代码？
- [ ] 是否指定了输出格式？
- [ ] 是否有矛盾或不合理的要求？
- [ ] 复杂任务是否已拆分？
