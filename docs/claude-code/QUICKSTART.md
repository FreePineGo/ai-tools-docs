# Claude Code 快速开始

## 5 分钟快速上手

### 第一步：启动 Claude Code

```bash
# 进入项目目录
cd /path/to/your/project

# 启动交互式会话
claude
```

### 第二步：基本对话

启动后，你可以直接输入自然语言指令：

```
你好，请帮我创建一个简单的 Python Hello World 程序
```

### 第三步：查看和执行建议

Claude Code 会：
1. 分析你的请求
2. 生成代码或命令
3. 等待你的确认
4. 执行操作

## 常用场景示例

### 场景 1: 创建新项目

```
帮我创建一个 React + TypeScript 项目，包含以下功能：
- 用户登录页面
- 仪表盘
- API 集成
```

### 场景 2: 代码解释

```
请解释这个文件的功能：
@src/components/App.tsx
```

### 场景 3: Bug 修复

```
这个函数返回了错误的结果，请帮我调试：
@src/utils/calculator.ts

问题是：当输入负数时，计算结果不正确。
```

### 场景 4: 代码重构

```
请重构这个文件，提高可读性和性能：
@src/legacy/old-module.js

要求：
- 使用 ES6+ 语法
- 添加类型注解
- 拆分大函数
```

### 场景 5: 添加功能

```
请在用户模型中添加以下字段：
- phone (字符串)
- avatar (URL)
- lastLogin (日期)

同时更新相关的 API 和测试。
```

## 特殊语法

### 引用文件

```
@filename       # 引用单个文件
@folder/        # 引用整个文件夹
@*.ts           # 引用所有 TypeScript 文件
```

### 多文件操作

```
请比较 @file1.ts 和 @file2.ts 的差异
```

### 条件执行

```
如果测试通过，就部署到 staging 环境
```

## 会话管理

### 保存会话

```bash
# 导出当前会话
claude --export session.json
```

### 加载历史会话

```bash
# 从文件恢复
claude --import session.json
```

### 清除上下文

```
/clear  # 清除当前会话历史
```

## 实用技巧

### 技巧 1: 分步请求

与其一次性要求复杂功能，不如分步进行：

```
第一步：创建用户模型
第二步：添加验证逻辑
第三步：编写 API 端点
第四步：添加测试
```

### 技巧 2: 提供上下文

```
项目背景：这是一个电商平台的后端服务
技术栈：Node.js + Express + MongoDB
当前问题：用户登录接口响应慢
```

### 技巧 3: 指定输出格式

```
请用表格形式列出所有 API 端点
请用流程图描述数据流
请用代码块展示修改内容
```

### 技巧 4: 迭代优化

```
这个实现不错，但请：
1. 添加错误处理
2. 优化性能
3. 添加日志
```
