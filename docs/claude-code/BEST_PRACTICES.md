# Claude Code 最佳实践

## 提示词工程

### 编写有效提示

#### ✅ 好的提示

```text
请创建一个用户认证模块，要求：
- 使用 JWT 进行令牌管理
- 支持邮箱和密码登录
- 包含密码重置功能
- 添加速率限制防止暴力破解
- 使用 bcrypt 加密密码

技术栈：Node.js + Express + MongoDB
```

#### ❌ 差的提示

```text
做个登录功能
```

### 提示词结构

```text
1. 任务描述：清晰说明要做什么
2. 上下文信息：提供相关背景
3. 具体要求：列出详细需求
4. 技术约束：指定技术栈和限制
5. 输出格式：期望的交付形式
```

### 上下文管理

```
# 提供项目背景
这是一个电商平台的后端服务，已有用户和订单模块

# 指定文件范围
只修改 @src/auth/ 目录下的文件

# 说明依赖关系
这个改动会影响用户服务和订单服务
```

## 工作流程

### 迭代开发流程

```
1. 需求分析
   → 描述需求，确认理解一致

2. 方案设计
   → 讨论实现方案，评估选项

3. 逐步实现
   → 分模块实现，每步验证

4. 测试验证
   → 运行测试，修复问题

5. 代码审查
   → 检查质量，优化改进
```

### 大任务分解

```
❌ 不要：一次性要求完成整个项目

✅ 要：分步骤进行
第一步：创建项目结构和配置
第二步：实现核心数据模型
第三步：开发 API 端点
第四步：添加认证和授权
第五步：编写单元测试
第六步：集成测试和部署
```

## 代码质量

### 代码审查清单

让 Claude Code 检查：

```
请审查这段代码：
- [ ] 代码可读性
- [ ] 错误处理
- [ ] 性能问题
- [ ] 安全漏洞
- [ ] 测试覆盖
- [ ] 代码规范
- [ ] 文档完整性
```

### 重构建议

```
请重构这个模块，关注：
1. 降低复杂度（目标：圈复杂度 < 10）
2. 提高内聚性（相关功能分组）
3. 减少耦合（依赖注入）
4. 添加类型注解
5. 改进命名
```

## 安全实践

### 敏感信息处理

```
❌ 不要：
- 在代码中硬编码密钥
- 提交包含密码的文件
- 在日志中输出敏感数据

✅ 要：
- 使用环境变量
- 添加到 .gitignore
- 使用密钥管理服务
```

### 安全审查

```
请进行安全审查：
1. 检查 SQL 注入风险
2. 验证输入 sanitization
3. 检查认证授权逻辑
4. 审计依赖漏洞
5. 验证加密实现
```

## 常见问题处理

### 处理模糊需求

```
当需求不明确时：

1. 主动询问澄清问题
   "这个功能的目标用户是谁？"
   "期望的响应时间是多少？"

2. 提供选项供选择
   "方案 A: 快速实现，适合 MVP
    方案 B: 完整实现，考虑扩展性"

3. 确认假设
   "我假设需要支持 1000 并发，对吗？"
```

### 管理期望

```
明确说明限制：

"这个实现可以处理基本场景，但：
- 不支持分布式部署
- 需要额外工作来支持高可用
- 建议在生产环境前进行压力测试"
```

## 效率技巧

### 快捷键和别名

```bash
# 常用别名
alias cc='claude'
alias cc-new='claude --new-session'
alias cc-review='claude "请审查这个 PR"'

# 快速启动模板
cc-quick() {
  claude --new-session "$1" --export session.json
}
```
